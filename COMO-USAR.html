Guia de instalação

Como usar o Scholar Nexus

Sem instalar nada, sem programar. Só precisa de um navegador (Chrome, Edge, Firefox ou Safari) e de manter os ficheiros na pasta certa.

Importante — leia antes de começar O Scholar Nexus é uma pasta de ficheiros .html. Para funcionar, os ficheiros têm de estar na organização exacta indicada abaixo. Não mova nem apague nenhum ficheiro da pasta.
Passo a passo
1

Crie uma pasta chamada scholar-nexus

No seu computador (por exemplo no Ambiente de Trabalho ou em Documentos), crie uma pasta com esse nome exacto. Dentro dela, crie mais três subpastas: assets, dashboard e admin. Dentro de assets, crie uma pasta chamada images.

📁 scholar-nexus/
    📁 assets/
        📁 images/
    📁 dashboard/
    📁 admin/
2

Coloque cada ficheiro no lugar certo

Descarregue todos os ficheiros desta conversa e coloque-os exactamente como indicado abaixo. A cor verde indica os ficheiros principais.

📁 scholar-nexus/
    📄 index.html   ← ficheiro principal (pesquisa)
    📄 login.html
    📄 register.html
    📄 pricing.html
    📄 compare.html
    📄 manifest.json
    📄 sw.js
    📁 assets/
        📁 images/
            🖼️ icon-192.png
            🖼️ icon-512.png
    📁 dashboard/
        📄 index.html   ← painel do utilizador
    📁 admin/
        📄 index.html   ← painel do administrador
3

Abra o ficheiro index.html no navegador

Entre na pasta scholar-nexus, clique duas vezes em index.html. Vai abrir no seu navegador. Não precisa de internet para abrir — a plataforma abre directamente. A internet só é necessária para fazer pesquisas (para ir buscar os artigos às bases científicas).

🌐

Chrome / Edge

Funciona perfeitamente. Recomendado para melhor desempenho.

🦊

Firefox

Também funciona. Apenas o registo do Service Worker (PWA) pode não estar disponível.

Credenciais de acesso
🔑

Conta de Administrador

Ao abrir a plataforma pela primeira vez, é criada automaticamente uma conta de administrador. As credenciais foram entregues separadamente, num guia privado que não deve ser colocado dentro desta pasta nem publicado no GitHub. Guarde esse guia apenas no seu computador.

Importante Depois de entrar pela primeira vez como administrador, vá imediatamente a Painel → Definições e altere a palavra-passe.
👤

Criar uma conta de utilizador normal

Qualquer pessoa pode criar a sua própria conta clicando em "Acesso Gratuito" ou indo a register.html. A conta fica guardada no navegador do computador.

O que faz cada página
🔍

index.html — Pesquisa

A página principal. Pesquise por tema, autor, título ou DOI em 8 bases científicas em simultâneo.

🔐

login.html — Entrar

Para iniciar sessão com uma conta já existente.

📝

register.html — Registar

Para criar uma nova conta gratuita.

💳

pricing.html — Planos

Ver os planos de subscrição e simular um pagamento (demonstração).

⚖️

compare.html — Comparar

Comparar até 4 artigos lado a lado (score, citações, acesso aberto, etc.).

📊

dashboard/index.html — O meu painel

Favoritos, colecções, histórico, downloads e perfil do utilizador.

🛡️

admin/index.html — Administrador

Gestão de utilizadores, pagamentos, APIs, logs e configurações do sistema.

💡

Opções avançadas

Na página de pesquisa, expanda "Opções avançadas" para activar/desactivar resumo, BibTeX, Sci-Hub Africa, português, etc.

Problemas comuns
❓

A página abre mas não aparecem resultados

Verifique se tem ligação à internet. As pesquisas precisam de internet para ir buscar artigos às bases científicas (Semantic Scholar, OpenAlex, etc.). A plataforma em si abre sem internet, mas as pesquisas precisam de rede.

❓

O painel do dashboard ou admin diz "Sessão não encontrada"

Tem de iniciar sessão primeiro. Vá a login.html e entre com as credenciais do administrador (ou crie uma conta em register.html). Depois aceda ao painel.

❓

Os meus favoritos desapareceram

Os dados ficam guardados no navegador (localStorage). Se limpou o histórico do navegador ou usou outro navegador/computador, os dados não estão disponíveis. Use sempre o mesmo navegador no mesmo computador para manter os dados.

❓

A fonte CORE não devolve resultados

A API da CORE.ac.uk funciona sem chave mas com resultados limitados. As restantes 7 fontes (Semantic Scholar, OpenAlex, CrossRef, Europe PMC, PubMed, arXiv, DOAJ) continuam a funcionar normalmente. Não é um erro — é uma limitação conhecida da CORE em modo público.
